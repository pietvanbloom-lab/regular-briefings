Claude Code 2.1.166 ships a June 6 security fix: relayed SendMessage requests from other Claude sessions no longer carry user authority, plus a new fallbackModel setting

Tier 1 High confidence Anthropic

The June 6 point release hardens cross-session agent messaging: a receiver now refuses permission requests relayed via SendMessage from another Claude session, so a delegated agent cannot inherit the user's authority. The same release adds a fallbackModel setting that tries up to three models in order when the primary is overloaded, glob-pattern support in deny-rule tool-name positions, and fixes JetBrains terminal flicker on 2026.1+. A follow-on 2.1.167 added reliability bug fixes the same day.

Source: code.claude.com changelog, releasebot.io Claude Code, first seen 2026-06-06

Why this matters before others see it: As multiagent fan-out becomes the default (last week's dynamic workflows), the new attack surface is one agent spoofing the user's consent to another. Anthropic is closing that authority-confusion hole at the CLI level before frameworks standardize on agent-to-agent messaging. Watch for MCP and rival agent SDKs to adopt explicit non-delegable permission scopes within weeks, and for this to become a named requirement in enterprise agent security reviews.

Why this matters before others see it: Memory that mutates on its own shifts the moat from model quality to a durable personal data graph that gets stickier the longer you stay. Watch Anthropic and Google to answer with their own long-horizon memory, and expect "memory portability" and edit logs to become the next consumer-AI battleground.
