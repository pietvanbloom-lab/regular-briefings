Tier 1Agentic AI and MCPHigh impact

MCP publishes its stateless release candidate, seven days before the July 28 final

The Model Context Protocol team put out the release candidate for the 2026-07-28 specification on July 20, confirmed by maintainer David Soria Parra. The core change makes MCP stateless: no initialize handshake and no protocol-level session ID, so every request is self-contained and any request can reach any server instance behind a plain load balancer. Clients now send new Mcp-Method and Mcp-Name routing headers, letting gateways route and throttle on the operation without opening the request body. A new Extensions framework lands with the RC, moving Apps (interactive sandboxed HTML) and Tasks out of the core, while Enterprise-Managed Authorization stays in as the centralized identity-provider path. Source: modelcontextprotocol.io

Why it matters: This is the biggest change to the agent wiring standard since launch, and it targets the production gap where agents fail on infrastructure rather than model quality. Watch the July 28 final for any breaking change between RC and release, and watch whether the big MCP hosts (Anthropic, Microsoft, OpenAI) ship stateless-compatible servers in the same week.

Agentic AI and MCP

MCP Apps and Tasks become first-class extensions in the release candidate

The RC introduces an Extensions framework with two official extensions. MCP Apps (SEP-1865) lets servers ship interactive HTML rendered in a sandboxed host iframe, with UI actions routed through the same consent and audit path as direct tool calls. Tasks, which shipped experimentally in the 2025-11-25 spec, moves out of the core into an extension and is reshaped for the stateless model, so a server returns a task handle that clients drive with tasks/get, tasks/update, and tasks/cancel.
